# RISK_BRIDGE — 災害/ハザード語彙を外部リスク評価 API へ橋渡しする

fudosan-ontology の `chousa`（調査・法令制限）ドメインには、災害・ハザード関連の用語が
標準形で収録されています（浸水・洪水・内水・高潮・土砂災害警戒区域・液状化・津波・
地震・ハザードマップ 等）。この語彙は、外部の**災害リスク評価 API**と会話するための
共通語彙として使えます——両者が同じ言葉を使えば、リスク情報を物件説明や調査メモに
自然に統合できます。

本リポジトリ（OSS）は、その**接続点（フック）だけ**を提供します。実際のリスクスコア
取得は行いません。理ツール標準どおり、この OSS はネットワーク呼出をしないためです。

## 役割分担

| レイヤー | 責務 | 本リポジトリ |
|---|---|---|
| 語彙レイヤー（本 OSS） | 災害/ハザード用語の標準化・名寄せ、語彙の提供 | ✅ 提供する |
| 接続スタブ（本 OSS） | 住所の正規化 + 辞書由来のハザード語彙を返す | ✅ 提供する（`assess_risk`） |
| リスク評価レイヤー（別実装） | 住所 → 災害リスクスコアの算出・取得 | ❌ 含まない（外部 API） |

## 接続スタブ（`assess_risk`）

MCP ツール `assess_risk` は、住所を正規化し、辞書由来の災害/ハザード語彙を返します。
`risk_score` は常に `null`、`connected` は常に `false` です（ネットワーク呼出をしないため）。

```json
{
  "tool": "assess_risk",
  "input": "千代田区霞が関1-2-3",
  "normalized_address": "千代田区霞が関1-2-3",
  "risk_score": null,
  "connected": false,
  "hazard_vocabulary": ["ハザードマップ", "浸水想定区域", "土砂災害警戒区域", "..."]
}
```

## 実リスク評価 API へ接続する（利用者側の実装）

実スコアを取得したい場合は、この OSS の外側に薄い接続レイヤーを置きます。
辞書のハザード語彙で正規化した属性を、利用者が選んだ災害リスク評価 API に渡す形です。

```
# 利用者側の環境変数（本 OSS は読みません）
RISK_API_BASE   外部リスク評価 API のベース URL
RISK_API_KEY    その API の認証情報
```

接続レイヤーの責務:

1. 住所 → 緯度経度（ジオコーディングは利用者側 / 公的ジオコーダ等）
2. 外部リスク評価 API を呼び、災害種別ごとのスコアを取得
3. fudosan-ontology のハザード語彙でラベルを正規化し、結果を物件説明・調査メモへ統合

> この分離により、語彙（共有してよい公共財）と、リスク評価ロジック・認証情報・
> ネットワーク（利用者が管理すべきもの）を明確に切り離せます。

## 法的留意

災害リスクの説明は、宅地建物取引業法の重要事項説明（2020 年の水害ハザードマップ
説明義務化を含む）と関係します。本 OSS は語彙の標準化のみを担い、説明義務の充足や
リスクの確定判断は行いません。最終的な記載・判断は有資格者が行ってください。
