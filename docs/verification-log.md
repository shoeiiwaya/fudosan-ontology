# fudosan-ontology 検証ログ

法令出典・換算定数の確証記録。確実な条文のみ legal_source/コードに用い、不確実は null。

## legal_source の検証範囲（公開前の明確化・2026-06-22）

辞書エントリの `legal_source`（約 200 件・民法 / 建築基準法 / 不動産登記法 / 宅地建物取引業法 /
借地借家法 / 都市計画法 / 旅館業法 / 地方税法 等）は、v1 生成時に**出典の実在を確認した参考条文
ポインタ**であり、不動産実務で広く参照される主流条文を指す。ただし**最新性・正確性は保証せず、
法的助言ではない**（→ README「法的免責」）。

本ログで**独立に再確認している（＝確証済み）**のは、ハーネス（`ontology.py`）が実行時ロジックで
依拠する次の 2 法令のみ:

- **住宅宿泊事業法第2条第3項**（年間提供日数 180 日以内）
- **不動産の表示に関する公正競争規約**（広告上 1 畳 = 1.62㎡ 以上）

その他の `legal_source` は参考メタデータであり、法的効果を伴う利用の前に現行法令の原文と専門家
（弁護士・税理士・宅地建物取引士 等）への確認を要する。条文の精緻な確証は今後のコントリビュート
で本ログに追記して育てる。

## 2026-06-13 — ハーネス(ontology.py)新規実装に伴う検証

### 法令出典（domains/conditions.json）
- **住宅宿泊事業法第2条第3項**: 住宅宿泊事業の年間提供日数の上限は180日。
  → `民泊可` エントリ legal_source に採用。ハーネスでは民泊「可」を運用可否として確定せず
    Approval（要確認）に回す（管理規約・自治体条例・近隣同意・届出の有無で実際の可否が変わるため）。
- **不動産の表示に関する公正競争規約（不動産公正取引協議会連合会）**: 広告で用いる1畳は1.62㎡以上。
  → `帖` エントリ legal_source に採用。面積換算（帖→㎡）の係数 JO_SQM=1.62 の根拠。
  → `新築` は建築後1年未満かつ未入居（同規約）。`築浅` caution に注記（築浅自体に法定基準なし）。

### 換算定数
- **1坪 = 約3.305785㎡**（尺貫法。1坪=6尺×6尺=畳2枚）。TSUBO_TO_SQM=3.305785。
- **新耐震基準 = 1981年6月1日以降の建築確認**。築年数 caution に注記（築年数のみで耐震性判断不可）。
- **和暦→西暦換算**: 昭和元年=1926（base 1925+1）、平成元年=1989（base 1988+1）、令和元年=2019（base 2018+1）。
  ERA_BASE で base+n の式に統一。元年は n=1 として扱う。

### 確定しない事項（要確認/null）
- ペット可否・楽器可否・事務所/SOHO可否の「相談可」は無条件可ではない。種別/頭数/時間帯/用途地域/管理規約に依存。
- 地番/住居表示の判別は語の手掛かりベースのヒューリスティック。断定不可は Approval で人間確認に回す
  （住居表示実施区域の正確な判定には自治体の住居表示台帳照合が必要だが、本ツールはネットワーク禁止のため照合しない）。


## 2026-06-13 — partial 是正（MT-068 方位誤検出 / MT-305 辞書dedup）

coverage_matrix.json の partial 指摘2件を実バグ修正。統計的な推定コードは追加していない（換算・判定のみ）。

### MT-068 方位正規化の誤検出（確証ある言語的事実）
- バグ: 「向き/向/面」の文脈が無い文で全文走査に fallback し、地名の方位字を方位と誤検出していた
  （実測: 「東口徒歩5分」→東 / 「西新宿」→西 / 「北区」→北 / 「南阿佐ヶ谷駅」→南 / 「東京都」→東）。
- 是正: 全文 fallback を廃止。方位は採光面を表す文脈語に隣接した方位語のみ採用する。
  - 採用文脈: 方位＋「向き/向/面/採光/方角/バルコニー/ベランダ/開口」、ラベル「採光：/方角=」＋方位、
    採光名詞（バルコニー/ベランダ/開口部/居室/寝室/リビング/窓）＋助詞/区切り＋方位。
  - 方位語の直後が「京/阪/神/駅/口/区/市/町/村」で始まる場合は地名とみなし不採用（lookahead）。
  - 文脈が無い英字方位（SE/SW/NE/NW/N/S/E/W）は単独トークン時のみ採用。
- 根拠: 方位字（東西南北）は地名・出入口・行政区の語にも頻出する。広告/物件文で「方位」を意味するのは
  採光面を語る文脈に限られるため、文脈語隣接を必須化するのが言語的に妥当（外部データ・統計推定不要）。

### MT-305 辞書dedup・過剰提案（辞書構造の事実）
- バグ1: ontology.json には正規形へ読み/略称を括弧併記した項目（例『三為（さんため）』）があり、
  norm_key が括弧記号は除去するが中身は残すため、本文の素トークン『三為』が既知語照合に当たらず新語誤提案。
- 是正1: 辞書ロード時に各表記から「括弧前の本体」「括弧内の読み/略称」を分離して別キー登録
  （_alias_surface_forms）。『三為』『さんため』の双方で既知語照合に当たる。
- バグ2: 特約/段取/業者/契約/取得/連絡 等の常用語・既知複合語の素片が大量に新語誤提案され精度が低かった。
- 是正2: (a) 文法語＋不動産の一般語を STOPWORDS に追加、(b) 既知語の表層形(known_surfaces)に対する
  双方向部分一致で複合語の素片（『特約』⊂『ローン特約』、『業者』⊂『住宅宿泊管理業者』）を除外。
- いずれも辞書自体は変更せず、ハーネス側の照合ロジックのみ修正。新語提案は従来どおり常に Hold（人間判断）。
